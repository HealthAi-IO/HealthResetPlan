# 第一部分 App和小程序需求文档
## 1. 项目概述
### 1.1 项目背景
针对高血压、高血脂、肥胖人群，结合用户身高、体重、年龄、体检/血脂血压检查结果，智能化定制个性化饮食与运动计划；实现检查报告自动识别、计划一键生成、全流程智能提醒、数据实时监测、打卡溯源等功能，帮助用户规范饮食、规律运动、按时用药，科学管控身体健康，改善三高与肥胖问题。

### 1.2 项目目标
1. 实现用户基础信息、检查报告数据的快速录入与智能识别提取
2. 基于用户身体数据与健康指标，自动生成专属7天饮食+运动计划
3. 搭建全场景智能提醒体系，覆盖饮食、运动、用药、称重全环节
4. 完成体脂秤硬件对接，实现体重、体脂数据自动同步
5. 构建完整打卡机制，记录用户饮食、运动、用药、称重执行情况
6. 形成用户健康数据闭环，为计划优化提供数据支撑

### 1.3 适用人群
存在高血压、高血脂、肥胖问题，需要科学饮食、规律运动、按时用药的人群。

## 2. 需求总述
本App和小程序核心是**健康数据智能化管理+个性化计划定制+全流程提醒打卡**，用户通过录入基础信息、上传检查报告，系统自动分析健康指标，生成针对性周饮食、运动计划；并通过定时提醒、硬件联动、打卡留存，全程监督用户执行，同时沉淀用户健康数据，实现健康管理的可视化、规范化。

## 3. 功能需求
### 3.1 核心功能
#### 3.1.1 健康信息录入与识别
1. 基础信息手动录入：用户输入身高、体重、年龄、性别、既往病史（高血压、高血脂、肥胖确诊情况）、用药信息（药名、用药时间、用药剂量）。
2. 检查报告图片识别：用户上传血脂、血压、体检报告图片，系统通过OCR智能识别关键指标（血压值、总胆固醇、甘油三酯、高密度脂蛋白、低密度脂蛋白、血糖等），自动填充至对应数据栏，支持用户手动修改校正。
3. 信息保存与修改：支持用户保存个人健康档案，后续可随时修改基础信息、重新上传检查报告更新数据。

#### 3.1.2 个性化计划自动生成
1. 周饮食计划生成：基于用户年龄、身高、体重、BMI、血压血脂指标，定制每日三餐精准菜谱（包含食材、分量、烹饪方式、食用时间），贴合三高+肥胖人群低盐、低脂、低糖、高纤维、优质蛋白的饮食原则。
2. 周运动计划生成：结合用户身体状况、肥胖程度、年龄，生成每日运动方案（包含运动类型、运动时长、运动强度、动作要领、最佳运动时间），避免高强度运动带来的健康风险。
3. 计划调整：支持用户根据自身口味、时间安排，手动微调饮食、运动计划，系统同步更新提醒内容。

#### 3.1.3 智能提醒功能
1. 饮食提醒：饭前2小时推送当日本次饮食菜谱，明确饮食注意事项、食材分量、烹饪要求。
2. 运动提醒：运动前1小时推送当日运动计划，提醒运动时间、运动内容、注意事项。
3. 用药提醒：按照用户设置的用药时间，提前推送用药提醒，明确药名、用药剂量。
4. 称重提醒：每日固定时间（如晨起空腹）提醒用户称重，同步提示连接体脂秤获取数据。

#### 3.1.4 硬件对接与数据同步
1. 体脂秤连接：支持蓝牙对接市面主流体脂秤，自动获取体重、体脂率、BMI等数据，无需手动输入。
2. 手动录入兜底：未连接体脂秤时，支持用户手动填写称重数据。
3. 数据留存：自动保存每日体重、体脂数据，形成健康数据曲线。

#### 3.1.5 打卡功能
1. 饮食打卡：完成饮食后，用户点击打卡，可上传饮食照片留存记录。
2. 运动打卡：完成运动后，用户点击打卡，记录运动完成情况。
3. 用药打卡：完成用药后，用户点击打卡，标记用药状态。
4. 称重打卡：体重数据同步/手动录入后，自动完成称重打卡，无需手动操作。

#### 3.1.6 健康数据统计
1. 每日数据汇总：展示当日饮食、运动、用药、称重打卡完成情况。
2. 周/月数据统计：生成体重变化曲线、打卡完成率报表，直观展示健康改善情况。
3. 计划执行复盘：统计计划完成率，为后续计划优化提供依据。

### 3.2 细分功能
1. 用户注册登录：支持微信一键授权登录，无需单独注册，简化操作流程。
2. 个人中心：展示个人健康档案、打卡记录、历史计划、数据统计报表。
3. 计划查看：支持按天查看每日饮食、运动、用药详情，可切换周视图、日视图。
4. 消息中心：集中展示所有提醒消息、打卡记录、系统通知。
5. 意见反馈：用户可提交计划不合理、功能异常等反馈，后台及时处理。

## 4. 非功能需求
### 4.1 性能需求
1. 响应速度：页面加载时间≤2秒，报告识别时间≤5秒，计划生成时间≤10秒。
2. 并发能力：支持单用户全流程流畅操作，满足日常高频次提醒、打卡需求。
3. 稳定性：系统7×24小时稳定运行，提醒功能准时触发，无延迟、漏提醒情况。

### 4.2 易用性需求
1. 操作简洁：所有功能流程简化，中老年人可快速上手，无需复杂操作。
2. 界面清晰：页面布局简洁明了，按钮、文字醒目，提醒内容直观易懂。
3. 容错性：用户上传错误报告、录入错误信息时，系统给出明确提示，支持快速修改。

### 4.3 兼容性需求
1. 适配微信App和小程序全版本，支持iOS、Android系统手机。
2. 兼容主流蓝牙体脂秤设备，覆盖市面80%以上家用体脂秤。
3. 适配不同尺寸手机屏幕，显示无错乱、无遮挡。

### 4.4 安全性需求
1. 用户健康数据、个人信息加密存储，杜绝数据泄露。
2. 硬件连接采用安全加密协议，保障数据传输安全。
3. 不获取用户无关隐私信息，严格遵守微信App和小程序隐私规范。

### 4.5 可维护性需求
1. 系统支持后续功能迭代，可新增健康指标、计划模板、提醒规则。
2. 后台可灵活修改饮食、运动计划库，适配不同健康状况用户。

## 5. 数据需求
### 5.1 用户基础数据
姓名（微信昵称）、性别、年龄、身高、体重、BMI、联系方式、注册时间、健康档案。

### 5.2 健康检查数据
血压（收缩压、舒张压）、血脂四项（总胆固醇、甘油三酯、高密度脂蛋白、低密度脂蛋白）、血糖、检查报告图片、检查时间。

### 5.3 计划数据
周饮食计划（每日三餐食材、分量、时间、烹饪方式）、周运动计划（运动类型、时长、强度、时间）、用药计划（药名、剂量、时间）。

### 5.4 打卡数据
饮食打卡记录、运动打卡记录、用药打卡记录、称重数据、打卡时间、打卡照片。

### 5.5 提醒数据
提醒类型、提醒时间、提醒内容、触发状态、用户读取状态。

### 5.6 硬件数据
体脂秤设备信息、体重、体脂率、BMI、数据同步时间。

## 6. 用户角色与使用流程
### 6.1 用户角色
单一用户角色：普通使用者（三高+肥胖人群），所有功能面向个人用户开放。

### 6.2 核心使用流程
1. 注册登录：微信授权一键登录，进入App和小程序。
2. 信息录入：填写身高、体重、年龄等基础信息，上传血脂血压检查报告，系统自动识别指标，录入用药信息。
3. 计划生成：系统基于健康数据，自动生成周饮食、运动、用药计划。
4. 日常使用：接收饮食、运动、用药、称重提醒→执行对应行为→完成打卡→体重数据自动/手动同步→查看每日打卡情况。
5. 数据查看：进入个人中心，查看周/月健康数据、计划执行情况。
6. 信息更新：身体指标变化时，重新上传检查报告、修改基础信息，系统更新计划。

## 7. 接口需求
1. OCR图片识别接口：实现检查报告图片文字提取、健康指标识别。
2. 微信授权登录接口：实现微信一键登录。
3. 蓝牙连接接口：实现与体脂秤的蓝牙对接、数据传输。
4. 消息推送接口：实现App和小程序本地提醒、微信服务通知推送。
5. 数据存储接口：实现用户数据、打卡记录、计划数据的存储与读取。

## 8. 异常需求
1. 检查报告识别失败：系统提示“识别失败，请重新上传清晰图片或手动录入指标”。
2. 体脂秤连接失败：提示“连接失败，请检查蓝牙开启、设备电量，或手动录入体重”。
3. 网络异常：提醒用户检查网络，离线状态可查看已生成计划，联网后自动同步数据。
4. 提醒未触发：系统后台自动重试，同时通过微信服务通知补发提醒。
5. 用户信息缺失：生成计划前，提示用户完善基础信息、检查报告数据，无法跳过。

# 第二部分 App和小程序设计文档
## 1. 产品整体设计
### 1.1 产品定位
一款专为高血压、高血脂、肥胖人群打造的**个性化健康管理微信App和小程序**，集数据智能识别、计划定制、智能提醒、打卡监测、硬件联动于一体，实现全流程科学化健康管控。

### 1.2 产品核心亮点
1. 智能识别：无需手动录入繁杂检查指标，上传报告自动提取。
2. 精准定制：结合三高+肥胖特点，生成专属饮食、运动计划，拒绝通用模板。
3. 全时提醒：覆盖饮食、运动、用药、称重全场景，杜绝遗漏。
4. 硬件联动：体脂秤数据自动同步，健康监测更便捷。
5. 闭环管理：从数据录入→计划生成→执行提醒→打卡留存→数据复盘，形成完整健康管理闭环。

### 1.3 产品架构层级
1. 表现层：App和小程序前端页面，用户操作交互界面。
2. 业务逻辑层：处理计划生成、提醒触发、打卡逻辑、数据对接。
3. 数据访问层：用户数据、计划数据、打卡数据存储与读取。
4. 外部接口层：OCR识别、蓝牙连接、微信消息推送接口。

## 2. 系统架构设计
### 2.1 前端架构（微信App和小程序）
1. 页面模块：登录页、首页、信息录入页、报告识别页、计划详情页、提醒页、打卡页、个人中心页、数据统计页。
2. 核心组件：表单录入组件、图片上传组件、蓝牙连接组件、消息提醒组件、打卡组件、数据图表组件。
3. 技术栈：微信App和小程序原生开发、WXML+WXSS+JavaScript、微信App和小程序API。

### 2.2 后端架构
1. 服务端：采用云开发模式，依托微信云托管，实现数据存储、接口调用、逻辑处理。
2. 核心服务：用户管理服务、OCR识别服务、计划生成服务、提醒调度服务、硬件对接服务、打卡统计服务。
3. 数据库：云数据库，存储用户所有健康数据、计划、打卡、提醒信息。

## 3. 功能模块详细设计
### 3.1 用户管理模块
1. 微信授权登录：调用微信授权接口，获取用户昵称、头像，自动完成注册登录，生成唯一用户ID。
2. 个人信息管理：用户可修改基础信息、用药信息、健康档案，支持信息删除、重置。

### 3.2 健康数据采集模块
1. 手动录入：表单设计，包含身高、体重、年龄、性别、血压、血脂、用药等输入框，必填项标红提示。
2. 图片识别：支持拍照、相册上传报告图片，调用OCR接口识别文字，系统筛选血压、血脂等关键指标，自动填充至对应输入框，用户可手动校正。
3. 数据校验：对录入的血压、血脂数值进行范围校验，超出正常范围给出健康提示。

### 3.3 个性化计划生成模块
1. 计划算法：基于用户BMI、年龄、血压血脂超标情况，匹配对应的饮食、运动规则库：
    - 饮食：低盐、低脂、低糖、优质蛋白、高纤维，控制每日热量，三餐定时定量。
    - 运动：低强度、循序渐进，以有氧运动为主，适配三高、肥胖人群身体承受能力。
2. 计划模板：预设7天循环饮食、运动模板，根据用户数据自动调整食材、运动强度、时长。
3. 计划输出：按日展示三餐饮食、运动、用药详情，支持周计划一键查看、单天计划微调。

### 3.4 智能提醒模块
1. 提醒设置：系统默认饮食（7-8点早餐、12-13点午餐、17-18点晚餐）、运动、用药、称重时间，用户可自定义修改。
2. 提醒触发：采用App和小程序本地定时器+微信服务通知双重触发，确保提醒准时到达。
3. 提醒内容：明确饮食菜谱、运动内容、用药剂量、称重要求，简洁易懂。

### 3.5 硬件对接模块
1. 蓝牙配对：用户开启手机蓝牙，App和小程序自动搜索附近体脂秤，完成配对连接。
2. 数据传输：连接成功后，体脂秤称重数据自动传输至App和小程序，解析体重、体脂率、BMI等指标。
3. 连接管理：支持设备解绑、重新连接，连接状态实时展示。

### 3.6 打卡管理模块
1. 打卡入口：每个提醒消息附带打卡按钮，首页设置快捷打卡入口。
2. 打卡记录：打卡时间、完成状态自动保存，饮食打卡支持上传照片。
3. 未打卡提示：超时未打卡，系统标记未完成，给出提醒督促。

### 3.7 数据统计模块
1. 可视化图表：采用折线图展示体重、体脂变化，饼图展示打卡完成率。
2. 数据汇总：按日、周、月汇总计划执行情况，生成健康小结。
3. 数据导出：支持用户导出个人健康数据、打卡记录。

## 4. 页面流程与UI设计规范
### 4.1 核心页面流程
登录页→首页（提醒、快捷打卡）→健康信息录入页→报告识别页→计划生成页→每日计划详情页→打卡页→个人中心页→数据统计页。

### 4.2 页面设计规范
1. 配色：主色调采用蓝色、绿色（健康、清新、舒缓），辅助色为浅灰色，避免鲜艳刺眼颜色，适配中老年用户。
2. 字体：字体选用无衬线字体，标题文字≥16px，正文文字≥14px，按钮文字醒目。
3. 布局：采用简洁卡片式布局，留白充足，按钮尺寸适中，便于点击操作。
4. 交互：点击、滑动操作简单，弹窗提示清晰，无复杂交互逻辑，减少用户学习成本。

### 4.3 核心页面内容
1. 首页：展示当日待完成提醒、快捷打卡入口、今日计划摘要、体重数据。
2. 信息录入页：分步骤录入基础信息、检查指标、用药信息，引导式操作。
3. 计划页：按天展示三餐饮食、运动、用药，左右滑动切换日期。
4. 个人中心：健康档案、打卡记录、数据统计、设置、意见反馈。

## 5. 数据库设计
采用微信云数据库，创建以下数据表：
1. 用户表（user）：用户ID、昵称、头像、性别、年龄、身高、体重、BMI、创建时间。
2. 健康指标表（health_data）：数据ID、用户ID、收缩压、舒张压、总胆固醇、甘油三酯、高密度脂蛋白、低密度脂蛋白、血糖、报告图片、录入时间。
3. 用药表（medicine）：用药ID、用户ID、药名、剂量、用药时间、备注。
4. 计划表（plan）：计划ID、用户ID、周饮食计划、周运动计划、计划生成时间、生效周期。
5. 提醒表（remind）：提醒ID、用户ID、提醒类型、提醒时间、提醒内容、触发状态。
6. 打卡表（clock）：打卡ID、用户ID、打卡类型、打卡时间、打卡状态、相关照片、关联计划ID。
7. 体重数据表（weight）：数据ID、用户ID、体重、体脂率、BMI、数据来源（体脂秤/手动）、测量时间。

## 6. 接口设计
### 6.1 用户相关接口
1. 微信登录接口：获取用户微信信息，完成登录注册。
2. 用户信息修改接口：更新用户基础信息、健康档案。

### 6.2 健康数据接口
1. 报告上传识别接口：上传检查报告图片，返回识别后的健康指标。
2. 健康数据保存接口：将手动/识别的健康数据存入数据库。

### 6.3 计划接口
1. 计划生成接口：传入用户健康数据，返回个性化周饮食、运动计划。
2. 计划修改接口：支持用户修改计划内容，更新数据库计划数据。

### 6.4 提醒接口
1. 提醒设置接口：设置/修改提醒时间、类型。
2. 消息推送接口：触发提醒消息，推送至用户App和小程序。

### 6.5 硬件接口
1. 蓝牙搜索连接接口：搜索并连接体脂秤设备。
2. 体重数据同步接口：获取体脂秤数据，存入数据库。

### 6.6 打卡接口
1. 打卡提交接口：记录用户打卡信息。
2. 打卡记录查询接口：查询用户历史打卡数据。

### 6.7 数据统计接口
1. 健康数据统计接口：返回用户体重、打卡完成率等统计数据。

## 7. 提醒机制设计
### 7.1 提醒时间规则
1. 饮食提醒：早餐前2小时（5-6点）、午餐前2小时（10-11点）、晚餐前2小时（15-16点）。
2. 运动提醒：用户设定运动时间前1小时，默认推荐19-20点运动，18-19点提醒。
3. 用药提醒：用户自定义用药时间前10分钟触发。
4. 称重提醒：每日晨起7点，提醒空腹称重。

### 7.2 提醒方式
1. App和小程序本地弹窗提醒，震动提示。
2. 微信服务通知提醒，确保用户未打开App和小程序时也能收到。
3. 未读提醒重复推送：10分钟内未读取，补发一次提醒。

## 8. 硬件对接设计
### 8.1 对接设备
支持蓝牙4.0及以上的家用体脂秤，适配主流品牌（华为、小米、云麦等）。

### 8.2 对接流程
1. 用户开启手机蓝牙，进入App和小程序“体重监测”页面。
2. App和小程序自动搜索附近可连接体脂秤，显示设备列表。
3. 用户选择对应设备，完成配对绑定。
4. 用户站上体脂秤，设备测量完成后，数据通过蓝牙自动传输至App和小程序。
5. App和小程序解析数据，自动完成称重打卡，保存体重、体脂信息。

### 8.3 异常处理
1. 蓝牙未开启：提示用户开启手机蓝牙。
2. 设备无响应：提示用户检查设备电量、重新靠近手机。
3. 数据传输失败：提示用户重新称重，或手动录入数据。

## 9. 安全设计
### 9.1 数据安全
1. 用户所有健康数据加密存储，云数据库设置权限管控，仅用户本人可查看。
2. 检查报告图片、打卡照片仅用于个人查看，不对外公开、不泄露。

### 9.2 隐私安全
1. 仅获取微信昵称、头像等基础信息，不获取用户手机号、位置等无关隐私。
2. 硬件对接仅获取体重、体脂相关数据，不采集其他设备信息。
3. 支持用户注销账号，一键删除所有个人数据。

### 9.3 接口安全
1. 所有接口加入身份验证，防止非法调用。
2. OCR识别、数据传输采用加密协议，防止数据窃取。
